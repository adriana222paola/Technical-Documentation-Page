Introducción

¿Qué es una Pokédex?

La Pokédex es una herramienta digital inspirada en el universo Pokémon que permite registrar, visualizar y organizar información sobre distintas criaturas. En este proyecto, aprenderás a construir una Pokédex básica utilizando HTML y CSS, enfocándote en la estructura semántica, el diseño responsivo y las buenas prácticas de accesibilidad. Este ejercicio no solo te ayudará a mejorar tus habilidades de maquetación, sino que también te permitirá aplicar conceptos técnicos en un contexto divertido y visualmente estimulante.

A lo largo de esta documentación, exploraremos cómo crear tarjetas individuales para cada Pokémon, cómo estilizar la interfaz para que sea atractiva y funcional, y cómo preparar el proyecto para futuras integraciones con JavaScript o APIs externas. Aunque el contenido es ficticio, el enfoque técnico es real: aprenderás a construir una base sólida que puede adaptarse a proyectos más complejos en el futuro.

Este proyecto es ideal para quienes están comenzando en el desarrollo web y desean aplicar conceptos reales en un entorno creativo.

Además de aprender HTML y CSS, este ejercicio te prepara para trabajar con datos dinámicos y mejorar la accesibilidad de tus interfaces.

La Pokédex puede expandirse fácilmente para incluir filtros, búsquedas y animaciones.

    ¿Qué información se necesita?

  • - Nombre del Pokémon
  • - Tipo o categoría (por ejemplo: fuego, agua, planta)
  • - Estadísticas básicas (HP, ataque, defensa)
  • - Botón de interacción (por ejemplo: “Ver más” o “Capturar”)
  • - Habilidad especial (por ejemplo: Intimidación, Clorofila)
  • - Evoluciones disponibles (por ejemplo: Charmander → Charmeleon → Charizard)


Estructura HTML


Estructura Base

La Pokédex se compone de tarjetas individuales para cada Pokémon. Cada tarjeta incluye nombre, tipo, imagen y estadísticas. Estas tarjetas deben seguir una estructura semántica que facilite la lectura tanto para humanos como para tecnologías de asistencia.

Un ejemplo básico incluye:

  • <div class="pokemon-card">: contenedor principal
  • <h3>: nombre del Pokémon
  • <img>: imagen del Pokémon
  • <ul>: lista de estadísticas

Estilos con CSS


Estilizando la Pokédex

El diseño visual debe ser atractivo, pero también funcional. Aquí se recomienda usar Flexbox para distribuir las tarjetas y aplicar media queries para asegurar que se adapten a distintos tamaños de pantalla. Puedes añadir efectos 3D, sombras suaves o paletas temáticas según el tipo de Pokémon para enriquecer la experiencia visual. 


  .pokedex-container {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
  justify-content: center;
}

.pokemon-card {
  border: 2px solid #ccc;
  border-radius: 10px;
  padding: 1rem;
  width: 250px;
  background-color: #f8f8f8;
  transition: transform 0.3s ease;
}

.pokemon-card:hover {
  transform: scale(1.05);
}

@media (max-width: 600px) {
  .pokemon-card {
    width: 100%;
  }
}


Datos Dinámicos


Integración de Datos

Para hacer la Pokédex dinámica, puedes usar JavaScript para consumir datos desde una API como PokeAPI. Esto permite mostrar información real y actualizada.


fetch("https://pokeapi.co/api/v2/pokemon/1")
  .then(response => response.json())
  .then(data => {
    console.log(data.name); // "bulbasaur"
  });

Accesibilidad


Buenas Prácticas de Accesibilidad

Una Pokédex accesible garantiza que todos los usuarios puedan interactuar con ella.

  • <nav> para la barra de navegación
  • alt="" en imágenes
  • Asegura que todos los botones sean accesibles por teclado (tabindex, aria-label).
